# Licensing

This repository uses a split licensing policy so the reusable agent kit is easy to copy, while finished demo products still carry attribution.

## Core Agent Kit: CC0 1.0

The reusable kit materials are dedicated to the public domain under **Creative Commons Zero 1.0 Universal (CC0 1.0)**:

- `docs/`
- `specs/`
- `prompts/`
- `templates/`
- root `README.md`
- root `README.zh-CN.md`
- root release and QA checklists

CC0 legal code: <https://creativecommons.org/publicdomain/zero/1.0/legalcode>

CC0 summary: <https://creativecommons.org/publicdomain/zero/1.0/>

You may copy, modify, translate, and reuse these kit materials without attribution. Attribution is appreciated but not required.

## Demo Products And Samples: CC BY 4.0

Original finished-guide content and sample data are licensed under **Creative Commons Attribution 4.0 International (CC BY 4.0)** unless a file says otherwise:

- `products/*` guide Markdown and HTML text
- `products/*` playlist TSV files
- `products/*` reconstructed execution logs
- `samples/*` guide and playlist sample data
- `site/` guide text, playlist presentation text, and static ebook layout

CC BY 4.0 legal code: <https://creativecommons.org/licenses/by/4.0/legalcode>

CC BY 4.0 summary: <https://creativecommons.org/licenses/by/4.0/>

You may share and adapt this material for any purpose, including commercial use, as long as you give appropriate credit, link to the license, and indicate if changes were made.

## Product Package Prompts

Prompt files inside `products/*` are included to help recreate the demo playlist. They may be reused under CC0 1.0 like the core kit prompts.

## Third-Party And Generated Visual Assets

Third-party images, source links, platform metadata, album artwork, screenshots, and quoted references keep their original licenses and terms. They are not relicensed by this project.

Generated visual assets are licensed only as described in the relevant `ASSET_MANIFEST.tsv` row. For AI-generated images, the manifest must state the generator, rights basis, and whether any third-party reference was used.

Do not bundle a third-party visual asset in a release unless it has been reviewed under `specs/ASSET_MANIFEST_SPEC.md`.
